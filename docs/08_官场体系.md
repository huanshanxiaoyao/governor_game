# 08 — 官场体系 (Officialdom System)

## 1. 概述

官场体系模拟明朝从皇帝到县级的行政层级。每局游戏创建一个"平行宇宙"：历史人物（来自 `key_persons.json` 的112位明代人物）被随机分配到各级官位，获得随机化的游戏内名字，但保留历史原型的性格、政治观点和籍贯。

**显示格式**: "丁岩(徐阶)" — 游戏名(历史原型)

**核心设计原则**: 官员 = Agent。复用现有 `Agent` 模型，扩展 `ROLE_CHOICES`，所有官员均为 FULL tier（LLM驱动，可对话）。

## 2. 君主原型系统

每局随机选择一种君主原型，决定全局政治气候：

| 原型 | 代表人物 | 特点 |
|------|---------|------|
| 勤政型 (diligent) | 朱元璋、朱棣 | 高税收压力、高反腐、低党争、低自由度 |
| 怠政型 (delegating) | 嘉靖、万历 | 低税压、高党争、高自由度、低任人唯贤 |
| 中庸型 (moderate) | 仁宗、宣宗、弘治 | 各项居中，较为平衡 |

**治国系数** (存于 `MonarchProfile.attributes`):
- `tax_pressure` — 税收压力系数
- `corruption_risk` — 贪腐风险系数
- `faction_intensity` — 党争强度
- `player_freedom` — 施政自由度
- `meritocracy` — 能力主义 vs 站队主义

## 3. 行政架构（对齐 GDD 3.3）

### 中央
- **皇帝** (1人) — 最高权力，直接使用历史真名（不做匿名化）
- **内阁** (4人) — 首辅1 + 成员3，朝廷决策核心
- **六部** — 吏、户、礼、兵、刑、工
  - 吏部: 尚书1 + 侍郎6（居六部之首，主管考核升迁）
  - 户部: 尚书1 + 侍郎3
  - 礼部: 尚书1 + 侍郎3
  - 兵部: 尚书1 + 侍郎3
  - 刑部: 尚书1 + 侍郎3
  - 工部: 尚书1 + 侍郎3
- **都察院** (8人) — 三层结构
  - 左都御史 1人 (二品)
  - 左副都御史 1人 (五品)
  - 监察御史 6人 (七品)

### 地方
- **巡抚** (1人) — 省级长官
- **布政使** (1人/省) — 省级财政与民政
- **按察使** (1人/省) — 省级司法与监察
- **知府** (1人) — 府级，复用现有赵廷章Agent
- **知州** (1人) — 与知县平级，更繁荣
- **知县** — 玩家

### 总计: ~47 个官场 Agent + 现有知府

### 玩家主线职位（硬约束）
- 玩家可达职位：`知县 → 知州 → 知府 → 巡抚（可兼中央一级部门副职）→ 内阁成员 → 内阁首辅`
- 非玩家晋升轨道（固定为关键NPC）：布政使、按察使、六部、都察院言官、总督
- 说明：这些NPC职位不让玩家任职，但会持续影响玩家考核、政策压力、派系关系和事件走向

### 入阁机制（硬约束）
- 提名来源：君主提名或首辅提名
- 吏部尚书可阻拦提名（低概率），是否阻拦取决于其利益得失
- 入阁前置条件：重点省份巡抚经历 + 中央一级部门副职经历 + 朝中影响力与君主/首辅关系达标

## 4. 派系系统

每局按君主原型生成2-3个派系：

| 勤政型 | 怠政型 | 中庸型 |
|--------|--------|--------|
| 改革派 (圣眷70) | 阁臣派 (圣眷60) | 务实派 (圣眷60) |
| 保守派 (圣眷40) | 言官派 (圣眷40) | 清流派 (圣眷50) |
| — | 中间派 (圣眷50) | — |

- 派系存储于 `Faction` 模型
- 官员派系归属存于 `Agent.attributes.faction_name`
- 首辅自动归属圣眷最高的派系
- 其余官员随机分配

## 5. 人物生成

### 匿名化流程
1. 从 `key_persons.json` 按类别 (君主/文臣/武将/文臣武将) 匹配官位需求
2. **皇帝**: 直接使用历史真名 → `agent.name`，`source_name` 为空
3. **其他官员**: 生成随机游戏内名字 → `agent.name`，历史真名 → `agent.source_name`
4. 保留籍贯 → `attributes.hometown`
5. 政治观点 → `attributes.political_views`
6. 主要事迹 → `attributes.deeds`
7. **人物复用**: 当历史人物池耗尽时，同一人物可支撑多个游戏内官员（不同假名）

### 数值属性
约40位关键人物在 `CHARACTER_ATTRIBUTE_MAP` 中预映射了：
- 智力/魅力/忠诚度
- 五大性格维度 (openness, conscientiousness, agreeableness)
- 三维政治倾向 (reform_vs_tradition, people_vs_authority, pragmatic_vs_idealist)
- 声望三维 (scholarly, political, popular)

未预映射的人物使用默认值。

## 6. 全国地方官系

### 数据来源
行政区划数据来自 `xingzhengquhua.json`（明永乐年间），包含15个省级行政区（排除短暂存在的交趾）。

### 生成规则
- **省级**: 每省 巡抚(1) + 布政使(1) + 按察使(1) = 3人
- **府级**: 每府 知府(1)，仅 type=府 或 军民府，跳过直隶州
- **总计**: ~45省级 + ~142府级 = ~187 地方官员
- 使用 `bulk_create` 批量创建以提升性能
- 历史人物池(112人)不足时自动复用（同一人物不同假名）

### 玩家归属
每局游戏随机分配玩家到某省某府，记录在 `county_data.admin_location` 中。现有知府(赵廷章)归属该府。

### 前端展示
按省分组的可折叠面板，玩家所在省高亮并默认展开。每省显示: 巡抚 → 布政使/按察使 → 知府列表。

## 7. MVP范围

### 已实现
- ~240个官场Agent（含皇帝），全国省/府全覆盖
- 3种君主原型 + 治国系数
- 2-3个派系 + 领袖指定
- 完整上下级层级（中央+地方）
- 官场层级API + 前端展示Tab（按省折叠）
- 现有知府(赵廷章)无缝整合
- 玩家随机归属某省某府

### 延期功能
- 官员对玩家的考核评价
- 派系动态变化（圣眷涨落、派系分裂）
- 朝廷大事件（换阁、大案、改元）
- 官员间的互动与阴谋
- 升迁/降职系统
- 与官场官员的对话交互

## 8. 数据模型

### Agent 扩展
- 新增 `source_name` 字段（历史原型真名）
- 新增 12 个 `ROLE_CHOICES`（EMPEROR → PREFECT_PEER）
- `attributes` 中新增: `org`, `rank`, `faction_name`, `superior_agent_id`, `source_person_id`, `deeds`, `political_views`, `assessment_tendency`, `province`, `prefecture`

### MonarchProfile (新模型)
- `game` (OneToOne → GameState)
- `agent` (OneToOne → Agent, 皇帝)
- `archetype` — 勤政/怠政/中庸
- `attributes` — 治国系数 JSONField

### Faction (新模型)
- `game` (FK → GameState)
- `name` — 派系名称
- `ideology` — 意识形态 JSONField
- `leader` (FK → Agent)
- `imperial_favor` — 圣眷 0-100

### 官职配置
- 中央: `POSITION_SPECS` 静态配置约39个Agent
- 地方: 从 `xingzhengquhua.json` 动态生成约187个Agent

## 9. 关键岗位职责（任务驱动）

| 角色 | 史实职责映射（简化） | 游戏内季度任务 | 对玩家的直接抓手 |
|------|---------------------|---------------|------------------|
| 知府 | 统筹府内县政、催科、复核司法、报送政绩 | 下达县级年度指标、审核重点案件、年终初评 | 税赋压力、案件态度、考核评语 |
| 巡抚 | 一省军政民政总协调，主抓稳定与危机处置 | 设定省级施政风向、灾年资源调配、跨府督办 | 巡察问责、特批赈济、越级奖惩 |
| 内阁首辅 | 统筹票拟与朝局议程，协调六部与皇帝意志 | 发布年度中央议题、整合各部意见、定考核口径 | 政策大方向、朝局温度、升迁窗口 |
| 内阁成员 | 分担票拟、形成政策联盟与制衡 | 支持/反对议题、派系游说、联名评议 | 站队收益/风险、关系与人情债 |
| 吏部 | 官员考课、升降任免 | 形成考核模板、调整升迁阈值 | 升迁概率、留任/贬黜倾向 |
| 户部 | 户籍、税粮、仓储财政 | 制定税收任务、灾年缓征/追征策略 | 县域财政压力与容错空间 |
| 礼部 | 科举、学校、礼制与名分秩序 | 发布文教导向、科举风向事件 | 文教收益、士林口碑 |
| 兵部 | 军务、征调、兵役 | 征兵征粮任务、治安联动督办 | 人口/治安压力、紧急军务事件 |
| 刑部 | 司法复核、重案审理 | 重案复核抽查、冤案追责 | 司法风险、清名变化 |
| 工部 | 工程营造、水利道路 | 重点工程立项、验收与追责 | 基建机会、超支与延期风险 |
| 言官（都察院） | 监察百官、风闻言事、纠弹失职 | 触发弹劾/纠察事件、提供负面舆情 | 清名与政治风险波动 |
