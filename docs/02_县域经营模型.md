## 1. 县域总览

### 1.1 县域结构
```
县城（行政中心）
├── 村庄 ×12（MVP）
│   └── 各有：人口、耕地、地主士绅(Agent)、村民代表(Agent)、有无村塾
├── 集市 ×3（MVP）
│   └── 各有：商户、商业税收、商户代表(Agent)
├── 县学 ×1（固定，容量和教学水平可变）
├── 衙门（玩家办公地，含衙役）
└── 附近卫所（军事独立，非县管辖）
```

### 1.2 县域基础属性

| 属性 | 初始范围 | 说明 |
|------|---------|------|
| 总人口 | 3000-8000 | 各村在册村民人数之和（县级公开口径） |
| 总耕地（亩） | 5000-15000 | 在册耕地总和（村民在册耕地 + 地主在册土地） |
| 县库现金（两） | 200-600 | 可用于投资的行政资金 |
| 民心 | 40-60 | 百姓对官府的认可度（0-100） |
| 治安 | 40-70 | 地区安全程度（0-100） |
| 商业指数 | 20-50 | 商业活跃程度（0-100） |
| 文教指数 | 10-40 | 教育文化水平（0-100） |

### 1.3 开局难度（通过初始属性组合实现）

| 难度 | 人口 | 民心 | 治安 | 特征 |
|------|------|------|------|------|
| 简单 | 6000+ | 55+ | 60+ | 鱼米之乡，地主开明，少灾少难 |
| 普通 | 4000-6000 | 45-55 | 50-60 | 中部平原县，问题均衡 |
| 困难 | 3000-4000 | 35-45 | 40-50 | 边远贫困县，地主强势，灾害频发 |

## 2. 村庄模型

### 2.1 双账本结构（每村）
```json
{
  "name": "李家村",
  "peasant_ledger": {
    "registered_population": 500,   // 在册村民总人数
    "farmland": 520,                // 村民在册耕地（无隐匿地）
    "grain_surplus": 0,             // 村民总粮食盈余
    "monthly_consumption": 0,       // 村民月消耗
    "monthly_surplus": 0            // 村民月均余粮
  },
  "gentry_ledger": {
    "registered_population": 30,    // 地主在册人数（仅记录）
    "hidden_population": 70,        // 地主隐匿人数
    "registered_farmland": 280,     // 地主在册土地
    "hidden_farmland": 80,          // 地主隐匿土地
    "grain_surplus": 0              // 地主总粮食盈余（只计入地主财富）
  },
  "hidden_land_discovered": false,  // 该村隐匿土地事件是否已触发（每村一次）
  "land_ceiling": 968,              // 潜在土地上限，初始=(在册地+隐匿地)×1.1~1.4
  "morale": 50,                     // 民心
  "security": 55,                   // 治安
  "has_school": false,              // 有无村塾
  "village_elder": "agent_101",     // 村民代表Agent
  "gentry": "agent_201"             // 地主Agent
}
```

口径规则：
- 村承载能力上限、县级人均余粮、徭役折银税基，仅使用 `peasant_ledger`。
- 农业税税基 = 村民在册耕地 + 地主在册土地。
- 地主账本盈余仅作为地主财富积累，不并入县级人均余粮。

### 2.2 过度开发

每村有固定的 `land_ceiling`（建县时生成），当实际耕地（含隐匿土地）超过上限的90%时，洪灾和旱灾概率递增（详见 `06a` §2.5）。玩家开垦前若 `utilization > 85%`，顾问会弹窗预警，可选择忽略或等待考察（1个月后月报给出“适合/不适合开垦”）。

### 2.3 村庄年度产出与余粮
```
基础亩产 = 基准值(2两/亩) × 天气系数(0.3-1.2) × 水利加成(1.0-1.3)

村民粮食总收入 = 基础亩产 × peasant_ledger.farmland
地主粮食总收入 = 基础亩产 × (gentry_ledger.registered_farmland + gentry_ledger.hidden_farmland)

地主总粮食盈余 = 地主粮食总收入
              - 地主在册人数 × 人均消耗 × 3
              - 地主隐匿人数 × 人均消耗

县级人均余粮 = Σ(各村村民账本总粮食盈余) / Σ(各村村民在册人数)
```

### 2.4 地主士绅 Profile
```json
{
  "id": "agent_201",
  "type": "gentry",
  "land_pct": 0.35,
  "character": "贪婪",        // 贪婪/开明/中庸
  "court_connection": "省级", // 无/府级/省级/中央级
  "wealth": 500,             // 两
  "cooperation": 40          // 与县衙的配合度（0-100）
}
```

## 3. 集市模型

### 3.1 集市属性
```json
{
  "name": "东关集",
  "merchants": 15,
  "trade_index": 35,        // 商业发展指数
  "annual_tax": 80,         // 年商业税收（两）
  "merchant_rep": "agent_301" // 商户代表Agent
}
```

### 3.2 商业税收计算
```
商业税收 = 商户数 × 基准税额(5两) × 商业发展指数/50 × 税率系数
```

## 4. 士绅阶层

### 4.1 定义
拥有科举功名（秀才、举人、进士）或退休官员、官员家属，构成县里的"士绅阶层"。

### 4.2 土地占有
- 明初士绅占地约30%，明末可达70%
- 南方集中度大于北方，但北方收成低，流民比例一般更高
- MVP中由开局难度决定初始占比（简单30%/普通40%/困难55%）

### 4.3 地主Agent Profile
```json
{
  "id": "agent_201",
  "type": "gentry",
  "land_pct": 0.35,
  "character": "贪婪",           // 贪婪/开明/中庸
  "court_connection": "省级",    // 无/府级/省级/中央级
  "wealth": 500,
  "cooperation": 40,             // 与县衙的配合度（0-100）
  "political_ideology": {        // 政治理念
    "social_vs_people": 0.3,
    "centralize_vs_distribute": 0.4,
    "realistic_vs_ideal": 0.3
  }
}
```

### 4.4 士绅行为模式
- 天然想扩大土地和资源
- 拥有科举功名者享受部分免税特权（增加县域税收压力）
- 有官场关联的士绅被处理时，会触发链式事件（上级施压）
- 开明型士绅可能配合公益（修桥、赈灾），但通常附带条件

### 4.5 隐匿土地与户口

**隐匿土地**：
- 开局每村生成 `farmland × 5%~15%` 的隐匿土地，记入 `gentry_ledger.hidden_farmland`。
- 隐匿地不在册、不纳税、不参与村民承载力。
- 水利建设期间若衙役等级≥1，按民心概率触发举报事件（每村仅一次）；失败则进入强制丈量，通常查出真实隐匿地的50%~90%（受民心/衙役等级/知识属性影响）。

**隐匿户口**：
- 地主兼并（annexation）成功时，按兼并规模比例将在册村民转入 `gentry_ledger.hidden_population`。
- 被转移人口从在册户口中消失，不纳税、不服徭役。
- V1不提供找回隐匿户口手段。

## 5. 军事与治安

### 5.1 卫所制（MVP设定）
- 军队和地方行政完全独立
- 遇山贼、起义等问题，报告附近卫所，军队来负责
- 县令无法直接调动军队，但可以请求支援（需要时间，受知府支持度影响）

### 5.2 县域衙役
- 类似现代辅警，县令可直接管辖
- 增设衙役增加行政开支，但提升治安
- 衙役效率受统御力影响

## 6. 经济循环

### 4.1 县年度收入
| 来源 | 计算方式 | 备注 |
|------|---------|------|
| 农业税 | 各村产出总和 × 税率(约10%-15%) | 中央规定基准，玩家可微调执行力度 |
| 徭役折银 | 在册村民人数 × 人均徭役折银额 | 仅在册村民；地主家默认不交徭役折银（含地主在册人数） |
| 商业税 | 各集市商业税收之和 | |
| 上级拨款 | 特殊情况申请（灾害、军事等） | 需要知府批准，受关系影响 |
| 地主摊派 | 灾年或工程时向士绅征收 | 高风险操作 |

### 4.2 县年度支出
| 项目 | 金额范围 | 说明 |
|------|---------|------|
| 上缴税额 | 总税收的60%-70% | 中央规定，可申请减免 |
| 行政开支 | 110-200两/年（基础） | 含官员俸禄、县丞俸禄、师爷束脩、六房书办、衙役饷银、县学经费、衙署杂费共7项；增设衙役/建成村塾时永久增加（明细见06a §4.5） |
| 投资支出 | 玩家决定 | 水利、教育、开垦等 |
| 突发支出 | 0-200两 | 赈灾、剿匪配合等 |

### 4.3 资金约束
- 县库不可为负；资金耗尽时进入"亏空"状态
- 亏空状态：无法进行投资行动，民心每季度-3，考核直接扣分
- 可向知府申请借调（消耗好感度），或向地主借款（产生利益绑定）

## 7. 投资与建设

> **物价指数缩放**：下表中的花费为基准值，实际成本 = `round(基准花费 × price_index)`。
> 各县域类型的 `price_index` 见 `06a_县域经营数值.md` §1.4。

### 5.1 投资项目清单（MVP）

| 项目 | 花费（两） | AP消耗 | 见效时间 | 效果 |
|------|-----------|--------|---------|------|
| 开垦荒地 | 50/次 | 1 | 当年秋 | 耕地+200亩，但3年后可能水土流失 |
| 修建水利 | 按耕地规模 | 2 | 2年后 | 全县亩产+20%，灾害抵抗+15%。费用=round(20×总耕地/10000×物价指数) |
| 扩建县学 | 80/次 | 1 | 2年后 | 文教+10，长期增加本地士子 |
| 资助村塾 | 30/村 | 1 | 1年后 | 目标村文教+5，民心+5 |
| 增设衙役 | 40/年 | 1 | 即时 | 治安+8，但增加年度行政开支 |
| 修缮道路 | 60/次 | 1 | 下季度 | 商业+8，集市税收+10% |
| 开设义仓 | 70/次 | 1 | 即时 | 灾害人口损失×0.65（不影响秋收减产），民心+5 |

### 5.2 投资效果衰减
- 水利设施每3年需维护一次（花费30两），否则效果归零
- 村塾每年需补贴（10两/村），否则停办
- 衙役为持续性支出

## 8. 核心指标联动

### 6.1 指标联动矩阵

| 动因 → 结果 ↓ | 民心低 | 治安低 | 商业低 | 文教低 | 税赋重 |
|---------------|--------|--------|--------|--------|--------|
| 民心 | — | ↓ | - | - | ↓↓ |
| 治安 | ↓ | — | ↓ | - | ↓ |
| 流民概率 | ↑ | ↑↑ | - | - | ↑ |
| 商业 | ↓ | ↓↓ | — | - | ↓ |
| 人口增长 | ↓ | ↓ | - | - | ↓ |

### 6.2 关键阈值
- 民心 < 25：触发"民怨沸腾"，随机村庄暴动概率每季度30%
- 治安 < 20：触发"匪患四起"，山贼事件每季度必触发
- 商业 > 70：触发"百业兴旺"，吸引外来商户，但也引来上级关注（可能加税）
- 文教 > 60：每年有概率产出"本县才子中举"事件，大幅提升声望

## 9. 季度结算流程

```
春季结算：
  - 投资项目进度更新
  - 人口自然增长/流失计算
  
夏季结算：
  - 灾害判定（旱灾/水灾/蝗灾）
  - 灾害损失计算
  - 治安事件判定

秋季结算：
  - 农业产出计算（基础产量 × 天气 × 水利 × 灾害影响）
  - 税收征收
  - 商业税结算

冬季结算：
  - 年度指标变化汇总
  - 人事考核
  - 投资项目到期检查
  - 下年度中央税额通知
```
