## 1. 县域指标定义

### 1.1 状态指标（0-100）

| 指标 | 含义 | 变化机制 | 关键阈值 |
|------|------|---------|---------|
| 民心 | 百姓对官府的认可度 | 每月结算：基础衰减 + 文教回升 + 重税惩罚（详见§5.1） | <25暴动风险，>75减少治安事件 |
| 治安 | 地区安全程度 | 每月结算：基础衰减 + 衙役加成 + 民心联动（详见§5.2） | <20匪患频发、人口流失，>70吸引商户 |
| 商业 | 商业发展水平 | 仅受道路修缮（+）和灾害（-）影响，在GMV计算中作为单商户基础贸易额 | >60触发商户增长，<25触发商户流失 |
| 文教 | 教育文化水平 | 通过县学和兴建村塾升级提升 | >60概率出才子，>80成为文教名县 |

### 1.1b 基础设施等级

三大基建（县学、水利、医疗）共享统一的等级体系：

| 规则 | 说明 |
|------|------|
| 等级范围 | 0→1→2→3（最高3级） |
| 分期建设 | 建设期间不能继续升级同类设施 |
| 维护翻倍 | 每提升1级，年度维护费用翻一倍：L1=M, L2=2M, L3=4M |
| 投资翻倍 | 每次升级投资费用翻倍：0→1=C, 1→2=2C, 2→3=4C |

| 基建 | 初始等级(财赋/宗族) | 初始等级(沿海/灾荒) | 效果 |
|------|----|----|-----|
| 县学 | 1 | 1 | 每级文教+10 |
| 水利 | 1 | 0 | 每级产出+15%，灾害减损15%/30%/60% |
| 医疗 | 1 | 0 | 疫病概率/严重度 ×0.85^级，人口增长 ×1.05^级 |

**投资费用与工期**：

| 基建 | 费用缩放因子 | 基础投资(C) | 基础维护(M) | 工期(级1/级2/级3) |
|------|------------|-----------|-----------|----------------|
| 县学 | price_index | 80×PI | 15×PI | 2/3/5月 |
| 水利 | 耕地×price_index | 20×F×PI | 10×F×PI | 8/12/18月 |
| 医疗 | 人口×price_index | 12×P×PI | 5×P×PI | 2/3/5月 |

F = 总耕地/10000, P = 总人口/1000, PI = price_index

### 1.2 经济指标

| 指标 | 单位 | 说明 |
|------|------|------|
| 人口 | 人 | 各村合计（在册民户），年度更新（九月） |
| 耕地面积 | 亩 | 含自耕地和地主占地 |
| 地主占地比 | % | 地主/宗族/绅衿占有的土地比例，影响徭役税基 |
| 县库现金 | 两 | 可用资金，负数代表欠债 |
| 集市GMV | 两/月 | 月贸易总额 = 商户数 × commercial × 需求系数，即时计算 |
| 年度农业产出 | 两 | 秋季计算 |
| 年度税收 | 两 | 农业税 + 徭役折银 + 商业税（分时征收，秋季汇总） |
| 年度支出 | 两 | 行政 + 医疗 + 投资 + 上缴 |

### 1.3 县域类型

游戏开局随机选取一种县域类型，基于类型设定初始参数（详见§6）。

| 类型 | 分布地域 | 核心矛盾 |
|------|---------|---------|
| 财赋核心型 | 江南太湖平原（苏、松、常、杭、嘉、湖） | 高额赋税定额与地主占地集中的冲突，徭役负担集中在少数平民 |
| 宗族治理型 | 皖南徽州、赣东南抚州等山区 | 宗族势力与官府权力的博弈，宗族主导基层秩序 |
| 沿海治理型 | 闽、浙、粤沿海 | 海防安全与民生发展的平衡，财政紧张 |
| 黄淮灾荒型 | 黄河、淮河中下游沿线 | 灾荒频发与固定赋税定额的冲突，流民问题突出 |

**类型对gameplay的影响**：
- **财赋核心型**：钱多事也多，上缴压力大，地主占地比高导致徭役税基窄、平民负担重
- **宗族治理型**：社会秩序稳定，税收征收效率高，但宗族势力强，改革阻力大
- **沿海治理型**：人少地少钱少，海防开支大，收支勉强持平，容错空间极小
- **黄淮灾荒型**：耕地多但灾害频繁，水患风险极高，需要持续投入水利和赈灾

### 1.4 物价指数 (Price Index)

每种县域类型有一个固定的物价指数（`price_index`），反映当地物价水平对施政成本的影响。

| 县域类型 | price_index | 说明 |
|---------|-------------|------|
| 财赋核心型 | 1.4 | 江南物价高，一切投资和开支都贵 |
| 宗族治理型 | 1.1 | 接近基准物价 |
| 沿海治理型 | 0.9 | 偏远地区物价稍低 |
| 黄淮灾荒型 | 0.8 | 中部物价较低 |

**物价指数影响范围**：
- **投资项目成本**：所有投资类型的花费按 `round(基础cost × price_index × 缩放因子)` 缩放
- **行政开支动态增量**：增设衙役 `+round(40 × price_index)`，村塾建成 `+round(10 × price_index)`
- **基建维护**：基础设施维护费用均含 `price_index` 缩放（见§1.1b、§4.5）

**物价指数不影响**：
- **基础行政开支（admin_cost_detail 前7项初始值）**：各县域类型的7项基础开支已按地区差异化设定（合计200/150/110/130两），隐含了物价水平
- **税收收入**：农业税、徭役折银、商业税均为产出或人口驱动，不受本地物价影响
- **农业产出**：亩产为实物产出折算，独立于物价

**设计意图**：物价指数为县域类型增加一层经济差异化——
- 财赋核心型（1.4）：投资成本高40%，加剧"钱多但花得更多"的困境
- 沿海治理型（0.9）：投资略便宜，部分补偿其极度紧张的财政
- 黄淮灾荒型（0.8）：投资便宜20%，为灾后重建降低门槛

**后续可扩展**（MVP不实现）：
- 动态物价波动：`price_index` 可按 ±0.05/年 随商业指数和灾害变化
- 影响徭役折银：物价越高，折银额应越高
- 影响商业税基：商户在高物价地区营收更高

## 2. 土地与农业

### 2.1 农业产出

代码中存在两套产出模型，分别用于不同用途：

**税收产出模型**（`_autumn_settlement`，用于计算农业税，以两为单位）：
```
村庄税收产出 = 耕地面积 × 基础亩产 × 农业适宜度 × (1 + 水利加成) × (1 - 灾害损失)

基础亩产 = 0.5两/亩（代码值，对应设计值2两/亩 ÷ 耕地×4缩放）
农业适宜度 = 0.3 ~ 1.0（全局环境参数，二月漂移）
水利加成 = 水利等级 × 0.15，即 0/0.15/0.30/0.45
灾害损失 = 基础损失 × (1 - 水利减损率)（见§2.2）
```

**粮食产出模型**（`_compute_peasant_production`，用于计算农民粮食储备，以斤为单位）：
```
农民粮食产出 = Σ(村庄自耕地 × 亩产200斤 × 农业适宜度 × 水利乘数 × (1-税率))
             × (1 - 灾害损失)

自耕地 = 耕地 × (1 - 地主占地比)
水利乘数 = 1 + 水利等级 × 0.15，即 1.0/1.15/1.30/1.45
```

> **注意**：两套模型的水利加成已对齐为 +15%/级。
> 税收产出按全部耕地计算；粮食产出仅按自耕地计算并扣税。

### 2.2 灾害系统

灾害在六月判定，每年最多触发一种。

```
灾害判定表：
  洪灾: 概率 = max(0.02, 水患风险 × (1 - 水利等级 × 0.1))
        严重度 0.4~0.7，基础民心-10
  旱灾: 概率 = 0.15 × (1 - 农业适宜度)
        严重度 0.3~0.6，基础民心-8
  蝗灾: 概率 = 0.08
        严重度 0.2~0.4，民心-5
  疫病: 概率 = 0.05 × 医疗乘数
        严重度 (0.05~0.15) × 医疗乘数，基础民心-15

  医疗乘数 = 0.85^医疗等级（降低疫病概率和严重度）
  概率下限: 洪灾在水患风险>0时最低2%（不可消除风险）

水利对洪灾概率的影响（轻微降低，直接使用水患风险值）：
  水利等级0: 概率 = 水患风险
  水利等级1: 概率 = 水患风险 × 0.9
  水利等级2: 概率 = 水患风险 × 0.8
  水利等级3: 概率 = 水患风险 × 0.7
  ※ 不同县域类型有不同的水患风险基准值（如灾荒型0.45，财赋型0.25等）

水利对灾害损失的减免（仅限洪灾和旱灾）：
  水利减损率 = [0, 0.15, 0.30, 0.60]（按等级）
  实际损失 = 基础损失 × (1 - 水利减损率)
  示例：severity=0.5 的洪灾，水利等级2 → 实际损失 = 0.5 × 0.70 = 0.35
  ※ 蝗灾、疫病不受水利减损

灾害即时效果（六月判定时立即生效）：
  民心: 洪灾/旱灾 → 基础民心惩罚 × (1 - 水利等级 × 0.1)
        疫病 → 基础民心惩罚 × 0.85^(1+医疗等级)
        蝗灾 → 基础民心惩罚（无额外减免）
  商业: commercial -= round(3 + 7 × severity)
  疫病人口损失: 每个村庄减少 population × random(0.02, severity/5) 的人口

灾害持续效果（秋季结算时）：
  农业产出损失 = 产出 × severity × (1 - 水利减损率)（仅洪灾和旱灾有水利减损，蝗灾和疫病无减损）
  ※ 义仓与赈灾不影响秋收减产，仅水利影响农业损失
  人口损失: 每个村庄减少 population × random(0.02, severity/5) 的人口（所有灾害类型）
  义仓: 人口损失 × 0.65
  赈灾: 人口损失 × 0.65（与义仓可叠加）
  双重生效: 人口损失 × 0.65 × 0.65 = 0.4225
  赈灾额外效果: 民心+2
```

### 2.3 农民粮食盈余与集市GMV

集市GMV（月贸易总额）由商户数、商业发展水平和需求系数即时计算，不再使用累加式贸易指数。

```
集市GMV = 商户数 × commercial × 需求系数

需求系数 = clamp(1 + 月均余粮 / 20, 0.1, 2.0)
  月均余粮 = (当前储备 - 12 × 月度基础消耗) / 总人口 / 12
  ※ 使用固定12月视野（年化），不随距秋收月数变动

需求系数参照：
  月均余粮+20 → 需求系数2.0（封顶）
  月均余粮+10 → 需求系数1.5
  月均余粮  0 → 需求系数1.0（基线）
  月均余粮-10 → 需求系数0.5
  月均余粮-18 → 需求系数0.1（底线）
```

粮食储备机制：
```
粮食储备（peasant_grain_reserve）：
  初始化 = 年度农民产出 - 4个月消耗（游戏开局为正月，距上次收获4月）
  每月扣除 = 总人口 × 年消耗(300斤) / 12 × 过度消费乘数
  九月秋收 += 当年实际农民产出（含灾害减损）

前瞻盈余（建县初始化时计算一次，之后每月更新）：
  使用固定12月视野（年化），避免除以变动月数导致的季节性失真：
  年化消耗 = 12 × 月度基础消耗
  人均余粮 = (当前储备 - 年化消耗) / 总人口
  月均余粮 = 人均余粮 / 12

  距秋收月数仅用于前端展示 = (9 - 当月) % 12，若为0则取12

初始化时同步计算集市GMV（避免开局显示为0）：
  需求系数 = clamp(1 + 月均余粮/20, 0.1, 2.0)
  各集市GMV = 商户数 × commercial × 需求系数

结算顺序（关键）：
  九月：先秋收（粮食入库）→ 再计算商业（GMV/商税）
  其他月份：直接计算商业
  ※ 秋收在商业计算之前，确保九月的需求系数反映丰收后的充裕

过度消费机制（防止粮食无限积累）：
  当月均余粮 > 15斤时触发，二次方增长：
    excess_ratio = 月均余粮 / 15
    实际月消耗 = 基础消耗 × (1 + excess_ratio² × 0.1)
  示例：
    月均余粮 = 15斤 → 消耗 ×1.10（+10%）
    月均余粮 = 20斤 → 消耗 ×1.18（+18%）
    月均余粮 = 30斤 → 消耗 ×1.40（+40%）
    月均余粮 = 45斤 → 消耗 ×1.90（+90%）
  效果：丰收后消耗上升，但斜率较缓，避免过度“吃掉”高余粮

  典型年度周期（开局财赋核心型）：
    正月~四月: 月均余粮 6→0，需求系数 1.3→1.0（春季平稳）
    五月~八月: 月均余粮 -2→-8，需求系数 0.9→0.6（青黄不接）
    九月(秋收): 月均余粮跳升至~29，需求系数 2.0（丰收旺季）
    十月~腊月: 月均余粮 28→23，需求系数 2.0（秋冬富足）
```

**灾害对商业的直接冲击**：
灾害发生时（六月），商业值立即下降 `round(3 + 7 × severity)` 点。
- 洪灾（severity 0.4-0.7）：商业-6~-8
- 旱灾（severity 0.3-0.6）：商业-5~-7
- 蝗灾（severity 0.2-0.4）：商业-4~-6
- 疫病（severity 0.05-0.15）：商业-3~-4

commercial 下降 → GMV 下降（因 GMV = merchants × commercial × demand_factor）。

**道路修缮递减**：每次修路收益递减，`max(0, 8 - 已修次数)`。

## 3. 人口

### 3.1 人口变化

人口每年九月（秋季结算时）统一更新一次，按村庄逐个计算。

**承载力上限**（每村独立计算）：
```
承载力 = 自耕地 × 农业适宜度 × 亩产(200斤) × 水利乘数 × (1 - 税率) / 年消耗(300斤)

自耕地 = 耕地 × (1 - 地主占地比)
水利乘数 = 1 + 水利等级 × 0.15
```

**年度人口变化**（按村计算）：
```
自然增长率 = 基础增长率(1.5%) × 民心乘数 × 医疗乘数 × 承载力修正
  民心乘数 = 1.01^(村民心-50)  （民心50为中性，每高1点+1%，每低1点-1%）
  医疗乘数 = 1.05^医疗等级    （每级+5%增长）
  承载力修正:
    余量>0时: (余量/上限)^0.5  （越接近上限增长越慢）
    超载时:   余量/上限 × 2.0  （超载时加速下降）
增长率上下限: ±2.5%

自然增长 = 人口 × 增长率（取整）

跨县迁移（邻县竞争）：
  对每个邻县按四项指标两两对比：民心、治安、商业、文教
  差值定义：Δi = 本县指标i - 邻县指标i

  判定阈值：
    显著领先：Δi ≥ +15
    显著落后：Δi ≤ -15
    持平：|Δi| < 10

  触发“显著流入”条件（对该邻县）：
    - 显著领先维度数 >=1
    - 显著落后维度数 = 0
    - 其余维度不可出现“显著落后”（可为“持平”或“中间”）

  触发“显著流出”条件（对该邻县）：
    - 显著落后维度数 >=1
    - 显著领先维度数 = 0
    - 其余维度不可出现“显著领先”（可为“持平”或“中间”）

  流动强度（实现）：
    - 显著维度数=1：迁移率0.5%
    - 显著维度数=2：迁移率1.5%
    - 显著维度数=3：迁移率2.0%
    - 显著维度数=4：迁移率2.5%

  按邻县逐个计算后汇总：
    - 对邻县j触发流入时：inflow_j = 邻县j人口 × 迁移率
    - 对邻县j触发流出时：outflow_j = 本县人口 × 迁移率
    - 年度总流入 = Σ inflow_j
    - 年度总流出 = Σ outflow_j
    - 单方向总迁移量上限：本县人口 × 5%

  村级分配：
    - 将县级流入/流出总量按各村人口占比分配到村庄，再与自然增长合并结算

  无邻县数据时：跨县迁移 = 0

年度变化 = 自然增长 + 跨县净流入 - 跨县净流出
```

### 3.2 征兵影响 `[未实现]`
```
征兵强度 = 边患风险度 × 基准征调比例(2%)
征调人口 = 总人口 × 征兵强度
影响：
  人口减少 = 征调人口
  民心 -= 征调人口 / 总人口 × 30
  耕地产出 -= 征调人口 / 总人口 × 0.5（劳动力减少）
```

## 4. 财政体系

### 4.1 农业税（秋季征收）
```
农业税 = 总耕地产出 × 税率 × 征收效率

  总耕地产出 = Σ(村庄耕地 × 0.5两/亩 × 农业适宜度 × (1 + 水利等级×0.15))
              × (1 - 灾害损失)
  税率 = 中央基准(12%) ± 玩家调整（范围9%~15%）
  征收效率 = 0.7 + 0.3 × (民心/100)，即民心0→0.7，民心100→1.0

说明：地主和自耕农适用相同税率，对所有在册土地统一征收。
（地主隐匿土地属于独立事件机制，不在基础税收公式中体现。）
```

### 4.2 徭役折银（半年度征收）

徭役分正月和五月两次征收，每次征收半年额度，各自按上缴比例扣除后入库。

```
年度徭役折银 = 应役人口 × 人均徭役折银额

  应役人口 = 总人口 × (1 - 地主人口比例)
  地主人口比例 ≈ 地主占地比 × 0.12（地主人少地多，占总人口比例远低于占地比）
  人均徭役折银额 = 0.3两/人·年

征收时机：
  正月（moy=1）：征收半年额 = 应役人口 × 0.3 / 2
  五月（moy=5）：征收半年额 = 应役人口 × 0.3 / 2
  每次征收后立即按 remit_ratio 上缴，留存入库

说明：地主（绅衿）依制免除徭役。地主占地比越高，免役人口越多，
应役人口越少 → 徭役总收入越低，且剩余平民人均徭役负担越重。

示例：
  人口5000，地主占地比60% → 地主人口比例7.2% → 应役人口4640
  年度徭役折银 = 4640 × 0.3 = 1392两（正月696两 + 五月696两）
```

### 4.3 商业税（月度征收）

商业税按月征收，地方固定留存60%（不使用农业税的上缴比例）。

```
月度商业税 = 全县月GMV × 商税税率
  全县月GMV = Σ(各集市GMV)
  各集市GMV = 商户数 × commercial × 需求系数（即时计算，见§2.3）

商税税率 = 基准3%，可调范围1%~5%，步长0.5%
调整商税税率时立即影响民心：每+0.5%税率 → 民心-1

商税留存：地方固定留存60%，上缴40%（独立于 remit_ratio）
  月度入库 = 月度商业税 × 0.6
  （预留：后续可通过谈判争取100%留存）

商户数量（merchants）年度调整（九月）：
  commercial ≥ 60 且 merchants < 30 → merchants += 1
  commercial ≤ 25 且 merchants > 2  → merchants -= 1
```

### 4.4 财政汇总

各税种在不同时机征收，按各自的上缴规则处理：

```
年度总税收 = 农业税（秋季） + 徭役折银（正月+五月） + 商业税（每月）

上缴规则：
  农业税、徭役折银：按 remit_ratio 上缴（可通过知府谈判调整±5%）
  商业税：固定上缴40%（地方留存60%，独立于 remit_ratio）

秋季结算时汇总全年数据：
  总税收 = 农业税 + 年度累计徭役 + 年度累计商税
  总上缴 = 农业税上缴 + 徭役上缴 + 商税上缴
  秋季净变化 = 农业税留存 - 行政开支 - 基建维护
  （徭役和商税留存已在征收时入库，不重复计算）

年度累计数据存储在 county.fiscal_year 中，正月重置。
```

### 4.5 行政开支与基建维护

行政开支由7个基础项目 + 基建维护组成，存储在 `county_data.admin_cost_detail` 中：

| # | 项目 | 键名 | 说明 | 动态变化 |
|---|------|------|------|---------|
| 1 | 官员俸禄 | `official_salary` | 知县+典史等官员俸禄 | 固定 |
| 2 | 县丞俸禄 | `deputy_salary` | 县丞（佐贰官）俸禄 | 固定 |
| 3 | 师爷束脩 | `advisor_fee` | 知县私聘幕友费用 | 固定（预留升级接口） |
| 4 | 六房书办 | `clerks_cost` | 吏、户、礼、兵、刑、工六房书吏薪俸 | 固定 |
| 5 | 衙役饷银 | `bailiff_cost` | 衙役/差役饷银 | 每次增设衙役 +round(40×price_index) |
| 6 | 县学经费 | `school_cost` | 县学运营 + 村塾补贴 | 每建成村塾 +round(10×price_index) |
| 7 | 衙署杂费 | `office_cost` | 办公、邮驿、修缮等杂费 | 固定 |
| 8 | 水利维护 | `irrigation_maint` | 水利设施年度维护 | 按等级翻倍（见§1.1b） |
| 9 | 医疗维护 | `medical_maint` | 医疗设施年度维护 | 按等级翻倍（见§1.1b） |

**各县域类型初始值（两/年）：**

| 项目 | 财赋核心型 | 宗族治理型 | 沿海治理型 | 黄淮灾荒型 |
|------|-----------|-----------|-----------|-----------|
| 官员俸禄 | 50 | 40 | 30 | 35 |
| 县丞俸禄 | 25 | 20 | 15 | 18 |
| 师爷束脩 | 30 | 25 | 20 | 22 |
| 六房书办 | 30 | 20 | 15 | 18 |
| 衙役饷银 | 16 | 12 | 8 | 10 |
| 县学经费 | 24 | 18 | 10 | 15 |
| 衙署杂费 | 25 | 15 | 12 | 12 |
| 水利维护 | M×PI | M×PI | 0 | 0 |
| 医疗维护 | M×PI | M×PI | 0 | 0 |

> 水利/医疗维护初始值：财赋/宗族型初始等级1，故有1级维护费用；沿海/灾荒型初始等级0，无维护。
> 维护费用按§1.1b公式：M = 基础维护×缩放因子×2^(level-1)，具体值取决于初始耕地/人口。

```
行政开支 = sum(admin_cost_detail 各项)（含基建维护）

动态变化规则：
  增设衙役：bailiff_cost += round(40 × price_index)
    衙役等级上限 = 3（即最多可增设3次）
  村塾建成：school_cost += round(10 × price_index)
  基建升级完成：更新对应维护项（irrigation_maint / medical_maint）
    维护 = round(基础维护 × 缩放因子 × 2^(新等级-1))

扣除时机：每年秋季结算时一次性扣除
  秋季净变化 = 农业税留存 - admin_cost
  县库 += 秋季净变化

示例：
  宗族治理型（price_index=1.1），增设衙役1次
  bailiff_cost = 12 + round(40 × 1.1) = 12 + 44 = 56
  admin_cost = 150 + 44 = 194两/年（不含基建维护变动）
```

### 4.6 基础设施投资费用

基建升级通过"投资施政"进行，统一规则见§1.1b。具体费用按缩放因子动态计算：

```
投资费用 = round(基础投资(C) × 缩放因子 × 2^(目标等级-1))

缩放因子：
  县学: price_index
  水利: (总耕地/10000) × price_index
  医疗: (总人口/1000) × price_index

各县域类型 0→1 级升级费用（基础投资×缩放因子）：
  县学（C=80）：
    财赋: 80×1.4=112  宗族: 80×1.1=88  沿海: 80×0.9=72  灾荒: 80×0.8=64
  水利（C=20）：
    财赋: 20×6×1.4=168  宗族: 20×5.2×1.1=114  沿海: 20×2×0.9=36  灾荒: 20×4.8×0.8=77
  医疗（C=12）：
    财赋: 12×8×1.4=134  宗族: 12×6×1.1=79  沿海: 12×3×0.9=32  灾荒: 12×5×0.8=48

升级限制：
  同类设施建设期间不可再次升级
  最高3级
  需有足够县库资金

工期：见§1.1b（县学2/3/5月，水利8/12/18月，医疗2/3/5月）
```

## 5. 指标月度变化

### 5.1 民心变化

**即时税率调整效果**（调税时立即生效）：
```
民心变化 = (旧税率 - 新税率) × 100 × 3
  加税1%: 民心 -3
  减税1%: 民心 +3
  示例: 12% → 15% → 民心 -9
  示例: 15% → 11% → 民心 +12
  50%传导至各村民心
```

**月度结算民心变化**：
```
月度变化 = 基础衰减(-0.33)
          + 文教贡献(文教指数/60)
          + 治安联动(治安>60时 +0.5, 治安<30时 -0.5)
          - 重税惩罚(税率>15%时 -1/月)

年化等效：
  基础衰减: -4/年
  文教贡献: 文教50 → +10/年, 文教30 → +6/年
  治安联动: 高治安+6/年, 低治安-6/年
  重税惩罚: -12/年

县↔村同步：
  县级变化的50%传导到各村
  各村按人口权重加权平均后，与县级值混合（70%村均 + 30%县值）
```

> **其他影响民心的一次性效果**（非月度结算）：
> - 赈灾投资：+8（投资执行时立即生效）
> - 义仓建成：+5
> - 开垦完成：目标村+5
> - 村塾建成：目标村+5
> - 灾害发生：-5~-15（灾害判定时立即扣减）
> - 税率调整：见上

### 5.2 治安变化

**月度结算**：
```
月度变化 = 基础衰减(-0.33)
          + 衙役加成(衙役等级 × 0.67)
          + 民心联动(民心>60时 +0.33, 民心<30时 -0.67)

年化等效：
  基础衰减: -4/年
  衙役加成: 等级1→+8/年, 等级2→+16/年, 等级3→+24/年
  民心联动: 高民心+4/年, 低民心-8/年

县↔村同步：同民心（50%传导 + 70/30混合）
```

> **其他影响治安的一次性效果**（非月度结算）：
> - 增设衙役：+8（投资执行时立即生效）

### 5.3 商业值与集市GMV

commercial 代表商业发展水平，在 GMV 计算中直接作为单商户基础贸易额。
GMV 为即时计算值，不存在累加或收敛机制。

```
商业值(commercial)变化来源：

1. 灾害直接冲击（六月，若有灾害）：
   commercial -= round(3 + 7 × 灾害严重度)

2. 道路修缮（投资完成时）：
   commercial += max(0, 8 - 已修缮次数)
   首次+8，第二次+7，...第九次+0

集市GMV（即时计算，无累加）：
  GMV = 商户数 × commercial × 需求系数
  需求系数 = clamp(1 + 每月人均余粮/20, 0.1, 2.0)
  每月人均余粮 = ((当前储备 - 12×月度基础消耗) / 总人口) / 12

年度节奏（通过需求系数体现）：
  秋收后（十月-腊月）：余粮充裕 → 需求系数高 → GMV高
  年中（四月-六月）：储备消耗 → 需求系数下降
  灾害月（六月）：commercial下降 + 粮食预期恶化 → GMV双重下跌
  收获前（七月-八月）：余粮可能转负 → 需求系数低 → GMV低
```

## 附录：月度结算流程

每月按以下顺序执行（`settle_county` / `advance_season`）：

| 步骤 | 时机 | 内容 |
|------|------|------|
| 1 | 正月 | 重置年度财政累计（fiscal_year） |
| 2 | 二月 | 环境漂移（农业适宜度、水患风险、边患风险） |
| 3 | 每月 | 检查并应用已完成的投资 |
| 4 | 六月 | 灾害判定 |
| 5 | 每月 | 民心月度变化 |
| 6 | 每月 | 治安月度变化 |
| 6b | 每月 | 地主兼并事件检查（仅玩家路径） |
| 7 | 正月、五月 | 半年度徭役征收 |
| 8 | 九月 | 秋季结算（人口更新 + 秋收 + 商户调整 + 农业税 + 行政/基建维护扣除）※先于商业计算 |
| 9 | 每月 | 商业更新（粮食扣除→年化月均余粮→GMV即时计算→月度商税） |
| 10 | 腊月 | 年终快照 + 清除灾害状态 |
