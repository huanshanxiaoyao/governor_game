# 09 — 知府司法审判系统

> v0.1 — 初稿
>
> 本文档定义知府阶段的司法玩法，包括二审复审系统与百姓击鼓鸣冤系统。
> 与 02b §8（推官职能概述）相互补充，本文档为详细设计规格。
> 案件池参考：`docs/historical_materials/typical_law_cases.json`

---

## 1. 设计目标

### 1.1 核心体验

司法系统不是独立的"断案小游戏"，而是知府**用人识人、平衡政治关系、维护公正声誉**的综合决策场景。

| 设计维度 | 具体体现 |
|---------|---------|
| 信息不对称 | 卷宗由下属知县提交，可能失真、隐瞒或出于私利 |
| 道德张力 | 正义与人情、原则与政治之间的两难 |
| 长链后果 | 每一个判决都可能在 1~3 个月后产生连锁事件 |
| 能力信号 | 推官的判断质量反映其能力；玩家的决策风格影响声誉 |
| 历史厚重感 | 律法引用、案由措辞、后果类型均贴近明代历史实际 |

### 1.2 设计原则

- **推官分流，玩家聚焦**：案件数量多，但推官自动处理大部分；玩家只面对高价值/高风险的关键案件
- **三个决策，无最优解**：核准原判 / 驳回重审 / 提审改判，各有合理场景和代价，没有"正确答案"提示
- **卷宗即线索**：案件文本中埋入推官标注的疑点，不做手把手引导，允许玩家自行判断
- **后果有延迟**：多数负面后果不即时显现，在 1~3 个月后以"事件触发"形式出现

## 2. 二审复审系统

### 2.1 案件生成节奏

每个季度结束时（三月、六月、九月、腊月），系统对全府所有下辖县自动生成重案卷宗：

```
全府案件池（每季度）：
  各县按月结算中，随机抽取已处理案件作为卷宗素材
  每县生成 2 份卷宗 → 共 N×2 份（N 为下辖县数）
```

**分流规则**（在季度推进时自动执行）：

| 优先级 | 条件 | 处理方式 |
|--------|------|---------|
| 1 — 知府亲决 | 推官标记"关键疑点"，且案件难度≥进阶 | 推送给玩家，本月内要求决策 |
| 2 — 委托推官 | 推官标记了疑点，但玩家当月已有3件待决 | 推官自动裁定（结果有一定随机性） |
| 3 — 常规核准 | 推官判断无异议，案情简单 | 自动核准，不推给玩家 |

**玩家每季度实际面对的案件数**：1~3 件（由推官筛选后推送），不会超载。

**案件池构成约束**：
- 卷宗池必须同时包含“原判明显失当”“原判大体正确”“原判正确但程序可更严谨”三类案件。
- 设计目标是不让“提审改判”成为稳定最优解；部分案件中，最佳做法应是直接核准原判。

**总览入口对齐**：
- 府情总览页新增“待办事项提醒”区块。
- 当 `pending_judicial_cases` 非空时，总览显示“待审议司法案件”提醒，并标注案件数量。
- 该提醒只承担提示与跳转职责；案件正文阅读、决策与日志仍统一在“司法”Tab 内完成。

### 2.2 推官的角色

推官（府衙司法副手）在每季度扫描各县卷宗后，给出简报：

**推送格式（推官简报）**：
```
【推官 陈廷弼 季度司法简报 · 第一年三月】

本季度全府共受理重案 12 件，已常规核准 9 件。
以下 3 件卷宗存疑，请大人过目：

① 华亭县 · 田产纠纷杀人案（进阶）
   疑点：凶器创口与认定凶器不符；被告供词时间矛盾
   建议：驳回重审，要求华亭县重新核验凶器

② 祥符县 · 赈粮监守自盗案（新手）
   疑点：证人与原审官存在亲属关系；赃粮流向不明
   建议：提审改判，直接传讯被告当庭核对

③ 聊城县 · 私盐贩卖案（高难）
   疑点：运盐船规格与卷宗严重不符；疑似团伙案被当作散户处理
   建议：大人亲自决断（本案涉及跨县协查，推官权限不足）
```

**推官能力对建议质量的影响**：

| 推官廉洁度/能力 | 建议准确率 | 说明 |
|---------------|-----------|------|
| 高能高廉 | ~85% | 疑点识别准，建议合理 |
| 中等 | ~60% | 偶有漏判，有时过于保守 |
| 低能/贪腐 | ~35% | 可能刻意回避某些案件、帮知县包庇 |

> 玩家通过通判核账发现推官廉洁度低时，应警惕其司法简报的可靠性。

### 2.3 知府决策选项

对于推官推送的每件案件，玩家有五个选项：

---

#### 选项 A：核准原判
**含义**：认可县级初审结果，签发复核文书。

**适用场景**：证据链完整、量刑合理、推官无异议的案件。

**基础后果**：
- 该县知县好感度 +5
- 按察使：无变化（正常流程）
- 民心：无变化

**风险**：若案件实为错判，后续被按察司复查发现 → 触发"失察"记录（见 §5.2）

---

#### 选项 B：驳回重审
**含义**：以卷宗不完整或证据不足为由，发回县级重审，限期补充证据。

**适用场景**：有明确疑点但尚不确定，或希望给下属补救机会。

**基础后果**：
- 该县知县好感度 -5（被驳回有面子损失）
- 按察使好感度 +5（体现知府严谨态度）
- 民心：无变化
- 限定知县 15 日内重新提交，逾期可追责

**风险**：知县可能二次敷衍了事，或以"案情复杂"拖延。

---

#### 选项 C：提审改判
**含义**：调取案犯至府衙，由推官代表知府直接提审，重新裁决。

**适用场景**：疑点明确、怀疑知县枉法、证据明显矛盾。

**基础后果**（视提审结论分支）：
- 若推翻原判：
  - 民望 +15~40（视案件影响力）
  - 按察使好感度 +10~20
  - 责任知县好感度 -20，可触发弹劾流程
- 若维持原判（提审后发现原审无误）：
  - 无额外效果，但流程合规

**代价**：
- 消耗推官本季度1次行动力（每季度最多主持2次提审）
- 得罪相关利益方（知县背后的士绅/朝中同乡）

---

#### 选项 D：委托推官裁定
**含义**：将此案完全交给推官按其判断处理，知府不直接参与。

**适用场景**：玩家精力有限，案件重要性中等。

**后果**：
- 推官按其能力和廉洁度自主裁定（约60%~85%概率做出合理判决）
- 若推官裁定失误且后续被发现 → 知府承担连带"失察"责任，但程度较轻（玩家"已授权"）
- 推官好感度 +3（被信任感）

---

#### 选项 E：搁置缓议
**含义**：暂不作决定，要求推官继续收集情报，下个季度再定。

**适用场景**：案情复杂、情报不足，但尚未到法定期限。

**后果**：
- 案件进入下季度待决队列
- 若涉及在押犯人，每延迟一季度民心微降 -2
- 最多搁置2季度，否则系统强制触发"按察使催办"事件

---

### 2.4 结果分支：提审后的情节树

提审改判是最具戏剧性的选项，其结果分为三类：

**A. 案情属实，冤案昭雪**
- 被告翻供，揭露知县枉法真相
- 触发：弹劾知县 → 知县被调查/免职；追回贪污资产归府库
- 全府民心 +，该县民心大幅提升

**B. 案情属实，但系知县疏失（非故意）**
- 原审存在程序错误，但知县并非主观枉法
- 触发：知县写悔过书，考核减分但不免职；补偿被冤当事人
- 民心小幅提升；与该知县关系小幅受损

**C. 提审后维持原判**
- 提审发现原审无误，被告确有罪
- 下属知县好感度 +10（知府确认了他的判断）；推官专业度加一分评价
- 对已服刑者给予书面确认，无其他影响

## 3. 百姓击鼓鸣冤系统

### 3.1 触发机制

每年**五月**和**十月**各触发一次，共2次。触发时呈现为叙事事件，不可跳过。

> **历史背景**：明代府衙设有登闻鼓，百姓可越过县级直接向知府击鼓鸣冤，但越级申诉有严格的程序限制——须有初审记录，不得无事生非，否则击鼓者本人受"诬告反坐"处罚。

**生成规则**：
- 优先选取玩家辖下**近两季度有异常**的县（知县廉洁度低、治安差、或存在驳回案件未处理的县）
- 若无异常，从全府随机抽取
- 五月事件类型倾向：**冤狱人命**（春秋大审前夕）
- 十月事件类型倾向：**土地纠纷/苛政**（税赋征收季）

### 3.2 事件结构

**击鼓鸣冤事件格式**：

```
【登闻鼓事件 · 第X年五月】

鼓声未歇，门吏来报：府衙门前有一老妇击鼓，自称祥符县陈王氏，
其子陈阿牛以杀人罪系狱，待秋后行刑，老妇称其子被冤，
跪于府门三日未食。

随附：祥符县刑房卷宗摘要（已由推官初步翻阅）
推官陈廷弼批注：卷宗中被告认罪供词与案发时间有明显矛盾，
               且关键证人系知县王怀德妻舅，建议知府亲阅原始卷宗。

事件限时：本月内须给出回应，否则城中百姓聚集，民心受损。
```

### 3.3 决策选项

---

#### 选项 A：受理，亲自复审
**流程**：调取原始卷宗，进入"提审改判"流程（同 §2.3 选项C）

**后果**：
- 城中百姓民心 +5（知府愿意倾听）
- 可能翻案（见 §2.4 结果树）
- 拖延超过1个月→民心额外 -5

---

#### 选项 B：受理，转交推官处理
**流程**：推官主持复审，知府不直接介入。

**后果**：
- 城中民心 +3（官府受理）
- 推官按能力处理（较高概率合理裁决）
- 若推官裁定失误→知府仍承担部分责任

---

#### 选项 C：不予受理，原判维持
**条件**：需在文书中写明不受理原因（系统提供模板选择）

**后果**：
- 即时：城中民心 -10，鸣冤者滞留府门
- 10 日后触发后续事件（概率性）：
  - 40% — 鸣冤老妇在府门外绝食，引发民众聚集，民心再 -15
  - 30% — 按察司收到举报，派员暗访，知府被评定"漠视民瘼"
  - 20% — 事件自然平息（无后续）
  - 10% — 言官在朝中弹劾，触发"声誉压力"事件

---

#### 选项 D：驳回，要求先走县级正规申诉程序
**含义**：以"未完成逐级申诉"为由，令其返回祥符县重新申诉。

**后果**：
- 程序合规，无即时负面
- 若该县知县廉洁度低：老妇大概率被拒于县门外 → 3个月后回到府衙再次击鼓，民心 -8，且追加"按察使关注"标记
- 若该县知县廉洁度高：县级正常受理，事件自然结束

---

### 3.4 鸣冤事件的结案与记录

每次击鼓鸣冤事件，无论处理结果如何，均写入府志（`pref-tab-log`），格式：
```
第X年五月 · 登闻鼓 · 祥符县陈王氏鸣冤案
决断：受理，提审改判
结果：陈阿牛无罪释放，华亭县知县周文彬被弹劾
后续：全府民心+12，按察使好感度+15，华亭士绅好感度-20
```

## 4. 案件类型与卷宗结构

### 4.1 案件类型分类

| 类型 | 典型案由 | 主要考点 | 推送频率 |
|------|---------|---------|---------|
| 吏治贪腐类 | 赈粮监守自盗、税粮私吞 | 识别知县枉法动机；吏治与关系平衡 | 高 |
| 冤狱平反类 | 命案嫁祸、刑讯逼供 | 证据矛盾识别；对抗强势士绅 | 高 |
| 统筹治理类 | 跨县私盐、流民盗匪 | 多县协调；上下级资源调动 | 中 |
| 民事纠纷类 | 田产争讼、族产继承 | 律法与情理平衡；乡绅关系 | 中 |
| 刑事重案类 | 强盗劫杀、械斗伤亡 | 快速决断；治安与震慑 | 低 |

### 4.2 卷宗标准结构

每份卷宗包含以下固定字段（LLM生成时遵循此模板）：

```json
{
  "case_id": "唯一标识",
  "case_name": "案件名称",
  "difficulty": "新手/进阶/高难",
  "category": "类型",
  "source_county": "提交县",
  "source_season": "提交月份（game season值）",
  "dossier_text": "完整卷宗正文（含律法引用）",
  "attachments": ["附件列表"],
  "suspicion_markers": {
    "critical": ["核心疑点（推官必然标注）"],
    "secondary": ["次要疑点（高能推官才能发现）"]
  },
  "options": [
    {
      "action": "核准原判/驳回重审/提审改判",
      "immediate_effects": {},
      "chain_events": []
    }
  ],
  "drum_eligible": false, // 是否可作为击鼓鸣冤事件的素材
  "judgment_truth": "可选；ORIGINAL_VALID/...", // 内部平衡标签，不在前端直接展示
}
```

### 4.3 难度分级规则

| 难度 | 疑点特征 | 推官能发现的比例 | 错判代价 |
|------|---------|----------------|---------|
| 新手 | 疑点明显（证人关系、账目数字）| 中等推官即可 | 轻（罚俸、减分）|
| 进阶 | 疑点隐蔽（时间线矛盾、凶器细节）| 需高能推官 | 重（失察记录、声誉损失）|
| 高难 | 疑点需跨县信息验证（涉及邻县公文、隐性政治关系）| 推官大多无法独立发现 | 极重（仕途影响、弹劾风险）|

## 5. 后果结算系统

### 5.1 即时效果（判决当月）

| 效果类型 | 触发条件 | 数值范围 |
|---------|---------|---------|
| 民望变化 | 所有案件决策 | -30 ~ +40 |
| 知县好感度变化 | 所有案件决策 | -25 ~ +15 |
| 按察使好感度 | 有疑点的案件 | -10 ~ +20 |
| 府库变化 | 提审改判追回赃款 | 0 ~ +1000两 |
| 推官好感度 | 委托处理/采纳建议 | ±5 |

### 5.2 延迟事件链（1~3个月后触发）

延迟事件以独立事件形式出现，不在判决当月显现：

**A. 失察记录（核准了错判）**

```
触发条件：核准原判 × 案件实为冤案 + 被按察司发现（概率30%~60%）
延迟：1~2个月
事件文本：「按察使李大人行文，指出本府XX月核准之XX县XX案卷宗有重大疏漏，
         特飞报巡抚存档」
后果：
  - 仕途考核 -10分
  - 按察使好感度 -15
  - 解锁"失察"标记（累计2次则考核降级）
```

**B. 被包庇者后续作恶（严重失察）**

```
触发条件：核准了涉及贪腐知县的错判
延迟：2~3个月
事件文本：「祥符县常平仓再度发现粮食亏空，知县王怀德弃官潜逃，
         巡抚责令本府追缴亏空，府库须赔付 XXX 两」
后果：
  - 府库 -200~500两
  - 全府民心 -10
  - 巡抚好感度 -20
```

**C. 冤情昭雪的声誉扩散**

```
触发条件：成功提审改判冤案
延迟：1个月
事件文本：「华亭县四乡百姓闻知府大人为陈阿牛昭雪，奔走相告，
         各处庙宇自发立牌位，称颂大人如父母官」
后果：
  - 全府民心 +10（持续加成，1个月内有效）
  - 江南士林好感度 +15
  - 解锁专属称号"青天知府"（条件：任期内昭雪冤案≥3起）
```

**D. 得罪士绅集团的政治反弹**

```
触发条件：提审改判时触动了豪绅利益
延迟：2~3个月
事件文本：「华亭林氏联合在朝同乡，向都察院御史行文，
         弹劾大人'草率提审、破坏祖制'，言官已具本待奏」
后果：
  - 触发"政治压力"事件（可主动应对：上书自辩/请托朝中盟友/忍受）
  - 若不应对：考核主观分 -5，巡抚收到负面风评
```

**E. 知县被弹劾后的权力真空**

```
触发条件：弹劾知县成功，知县被撤职
延迟：1个月
事件文本：「祥符县知县王怀德已奉旨革职，县中由县丞署理，
         然该县吏事糜烂，急需大人提名新任知县」
后果：
  - 触发"人事建议"机会（可推荐府试门生担任，建立更深师生绑定）
  - 县丞署理期间，该县税收完成度 -15%，治安小幅下滑
  - 若推荐的人选后续表现优秀，吏部好感度提升
```

### 5.3 按察使专项巡察触发条件

以下行为会提高当年按察使抽查概率（基础概率15%）：

| 行为 | 概率增加 |
|------|---------|
| 当年核准疑点案件≥3件 | +20% |
| 当年击鼓鸣冤事件选择"不予受理" | +25% |
| 委托推官处理≥5件（偷懒标记）| +10% |
| 下属知县廉洁度普遍偏低 | +15% |
| 推官廉洁度低且委托处理较多 | +15% |

## 6. 与其他系统的联动

### 6.1 与知县 AI 系统的联动

| 联动点 | 说明 |
|-------|------|
| 知县廉洁度 | 决定其提交卷宗的诚实程度；低廉洁度知县更可能制造冤案 |

| 下属汇报准确性 | 司法腐败会拉低该县汇报中治安/民心档位的真实性 |

| 联动点 | 说明 |
|-------|------|

## 8. 案件池设计思路

### 8.1 初始固定案件库

参考 `typical_law_cases.json` 中的3个典型案件，扩展为覆盖5种类型的初始案件池（约15~20件），确保前2年有足够不重复的体验。

**初始池构成建议**：

| 类型 | 数量 | 难度分布 |
|------|------|---------|
| 吏治贪腐类 | 5件 | 新手×2，进阶×2，高难×1 |
| 冤狱平反类 | 5件 | 新手×1，进阶×2，高难×2 |
| 统筹治理类 | 4件 | 进阶×2，高难×2 |
| 民事纠纷类 | 3件 | 新手×2，进阶×1 |
| 刑事重案类 | 3件 | 新手×1，进阶×1，高难×1 |

### 8.2 程序化生成（v2扩展）

基于 LLM + 固定模板，动态生成新卷宗：
- 输入：府域类型、当前季度、案件类型、难度级别
- 输出：符合 §4.2 卷宗标准结构的 JSON
- 约束：律法引用须准确、疑点须逻辑自洽、涉及人物名字与府域文化相符

### 8.3 案件池消耗与刷新

- 已被玩家决策的案件不重复推送
- 若可用案件池不足（游戏后期），退化为生成相似案件的变体
- 某类案件集中出现，说明对应知县存在治理问题（可作为剧情线索）
