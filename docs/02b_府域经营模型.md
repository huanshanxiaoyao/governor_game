## 1. 设计目标

### 1.1 核心体验转变

| 维度 | 知县/知州（02） | 知府（本文档） |
|------|---------------|---------------|
| 管理方式 | 亲力亲为——直接管理村庄、集市 | 委托治理——通过下属知县/知州间接管理 |
| 核心挑战 | 资源配置与两难抉择 | 识人用人与信息判断 |
| 信息粒度 | 村级数据，月报详尽 | 县级汇报，经过下属过滤 |
| 政治压力 | 来自知府（1人） | 来自巡抚/布政使/按察使（多方） |
| 时间尺度 | 月度决策，事无巨细 | 季度决策，抓大放小 |
| 核心技能 | 经济治理、谈判 | 用人、协调、政治平衡 |

### 1.2 设计原则

- **委托而非替代**：玩家不能越级干预县级村庄事务，只能通过政策、指令、人事影响知县/知州行为
- **信息迷雾升级**：知县可能报喜不报忧，数据有延迟和偏差，需通过多渠道验证
- **规模不等于复杂**：管理更多单位不意味着更多微操，而是决策层级的跃迁
- **延续与传承**：玩家在知县/知州阶段治理过的县/州成为辖下之一，可对比 AI 知县的治理差异

## 2. 行政层级说明

### 2.1 县与州的关系

**游戏设定（简化）**：

```
府（知府）
├── 县 ×N（知县，七品）
└── 州 ×M（知州，六品）
```

- **县和州是同级行政单位**，均直属于府
- 州比县更繁荣——面积更大、人口更多、商业更发达
- 知州品级略高（六品 vs 七品），但治理模式与知县完全相同（复用 02 县域经营模型）
- **州不下辖县**——游戏简化，所有县和州平级隶属于府

> **历史说明**：明代行政区划中，州分为"直隶州"（直属布政司，可辖县）和"属州"（隶属于府，类似大县）。
> 游戏简化处理——所有州均为属州，不辖县，直接隶属于府。这避免了多层嵌套的复杂性，
> 同时保留了州与县的规模差异作为 gameplay 变量。

### 2.2 玩家职位路径（对齐 05 §5.1）

```
七品知县（县）→ 六品知州（州）→ 四品知府（府）→ ...
```

- **知县 → 知州**：平级晋升（升品级，管更繁荣的地区），玩法不变，复用 02 县域模型
- **知州 → 知府**：质变——从亲力亲为转为委托治理，本文档核心内容

### 2.3 知州阶段说明

知州使用与知县**完全相同**的经营模型（02 县域经营模型），区别仅在于：

| 参数 | 知县（县） | 知州（州） |
|------|-----------|-----------|
| 品级 | 七品 | 六品 |
| 辖区规模 | 4~6村、1~3集市 | 更大（具体参数待定） |
| 人口基数 | 3000~8000 | 更高 |
| 商业水平 | 按县域类型 | 偏高 |
| 上级期望 | 标准 | 更高（税赋定额更大） |
| 考核标准 | 知县标准 | 同知县标准，但阈值更高 |

> 知州的具体参数（村庄数、人口、商业等）将在后续迭代中定义，暂不展开。
> 核心机制完全复用 02，无需额外设计。

### 2.4 晋升条件（对齐 05 §5.3）

**知县 → 知州**：
- 考核达标（大考"升迁"评级）
- 无重大失职记录

**知州 → 知府**：
- 至少2地以上任职经历（含知县/知州任期和平调）
- 累计至少5年任职
- 无重大失职记录

## 3. 府域总览

### 3.1 府域结构

```
府城（行政中心）
├── 下辖县/州 ×5~8（按府域类型）
│   └── 各有：AI知县/知州、县域数据（完整 county_data）
│        └── 各县/州下辖：村庄、集市（AI自主管理，玩家不可直接干预）
├── 府学 ×1（等级制，1→3级，影响全府文教与府试质量）
├── 府衙（玩家办公地）
│   └── 幕僚：通判（财税副手）、推官（司法副手）
├── 府级仓储（府级义仓，应对跨县灾害）
├── 跨县驿道（等级制，影响信息传递速度和商业联通）
└── 府域驻军（卫所，由都指挥使管辖，非知府直辖）
```

**下辖县/州构成**：
- 一个典型的府下辖5~8个县和州的混合
- 其中1~2个为州（更繁荣），其余为县
- 玩家曾任职的县/州也在辖下（由 AI 知县/知州接管）

### 3.2 府域基础属性

| 属性 | 说明 |
|------|------|
| 总人口 | 各县/州人口之和（来自下属汇报，有偏差） |
| 府库现金（两） | 可用行政资金，来自税赋分成 |
| 均民心 | 各县/州民心加权平均（按人口权重） |
| 均治安 | 各县/州治安加权平均 |
| 府域商业 | 跨县贸易总量（由各县/州 commercial 和驿道等级驱动） |
| 府域文教 | 各县/州文教加权 + 府学加成 |
| 年度总税收 | 各县/州上缴总额 |
| 上缴完成率 | 实缴 ÷ 省级定额 |

### 3.3 府域类型

府域类型由下辖县/州的组成决定，影响核心矛盾和省级期望：

| 类型 | 典型 | 下辖构成 | 核心矛盾 |
|------|------|---------|---------|
| 财赋重府 | 松江府、苏州府 | 多为财赋核心型 + 1~2宗族型 | 高额税赋定额 vs 地方发展空间 |
| 边防要府 | 大同府、宣府 | 多为灾荒型 + 沿海型 | 军事安全 vs 民生投入 |
| 均衡内陆 | 南昌府、长沙府 | 混合型，各类均有 | 均衡发展 vs 突出重点 |
| 贫困边远 | 广西柳州府 | 多为沿海型 + 灾荒型 | 极度有限资源下的生存 |

## 4. 下辖县/州管理

### 4.1 AI 知县/知州系统

每个下辖县/州由一位 AI 知县或知州治理，复用现有 AI 知县系统（见 neighbor 系统）：

- **AI 决策**：AI 知县/知州每月独立决策投资、税率调整等（LLM + 规则回退）
- **物理结算**：使用相同的月度结算引擎（`settle_county`）
- **人格驱动**：5种执政风格（民本/政绩/保守/进取/圆滑），三层属性体系
- **记忆系统**：保留近8个月决策记忆，LLM 参考历史行为

**与 neighbor 系统的差异**：
- AI 知县/知州受玩家（知府）指令影响——政策方向、税赋定额、投资优先级
- 下属有独立的好感度系统——对玩家的服从度受好感度影响
- 玩家可以"约谈"（对话）下属，施压或协商

### 4.2 下属汇报

**汇报节奏**：每年固定四次，分别在**二月、五月、八月、十一月**，由下属知县/知州提交。内容包括：
- 各项指标状况（模糊显示，见下）
- 本期主要施政行动
- 特记事项（可能隐瞒负面事件）
- 税收完成进度

**信息模糊机制**：下属汇报不提供精确数值，所有关键指标以**8档状况词**显示。下属诚实度和能力影响档位的准确性（可能偏高1~2档）。

| 档位 | 对应区间（0-99） |
|------|----------------|
| 极差 | 0 ~ 12 |
| 差   | 13 ~ 24 |
| 稍差 | 25 ~ 37 |
| 勉强 | 38 ~ 49 |
| 及格 | 50 ~ 62 |
| 稍好 | 63 ~ 74 |
| 良好 | 75 ~ 87 |
| 优秀 | 88 ~ 99 |

**汇报示例**：
```
华亭县 · 张知县 · 五月汇报
  人口规模：良好      县库状况：稍好
  民心：及格 ↑        治安：稍差 →
  商业：良好 →        文教：勉强 ↑
  本期行动：扩建县学（第2级）
  特记：无
```

### 4.3 下辖县州总览页面

参照县域管理的村庄页面，府域管理设有**下辖县州总览页面**，供玩家随时查阅。

- 列出全府所有下辖县/州，每行展示：县州名、当任知县/知州、各核心指标的档位状况
- 指标档位来源于最近一次汇报，**非实时数据**，两次汇报之间数据不刷新
- 可点击进入单县详情，查看该次汇报的完整内容及历史趋势（仍为档位，不显示精确值）
- 通判核账、推官巡查、暗访等操作的结果，仅在当次核查后临时显示精确值，不持久写入页面

### 4.4 信息验证

玩家可通过以下方式突破档位模糊，获取更准确的信息：

| 方式 | 效果 | 代价 |
|------|------|------|
| 通判核账 | 财务相关档位显示精确值，偏差±5% | 每年最多对3县/州执行 |
| 推官巡查 | 治安/司法相关档位显示精确值，偏差±5% | 每年最多对3县/州执行 |
| 暗访（亲临） | 获取全部指标精确值 + 触发随机发现 | 消耗1月行动力，下属若发现则关系-10 |
| 耳目情报 | 获得模糊预警（"某县民心可能不实"） | 持续消耗（府库开支） |
| 下属举报 | 其他知县/知州举报某县问题 | 需验证真伪，可能是构陷 |

### 4.5 干预手段

玩家对下属县/州的干预从间接到直接，代价递增：

| 手段 | 描述 | 效果 | 后果 |
|------|------|------|------|
| 政策建议 | 传达施政方向建议 | 下属可能采纳（取决于好感度和认同度） | 无负面 |
| 下达指令 | 明确要求执行特定行动 | 下属通常服从，但效果取决于能力 | 低好感下属可能阳奉阴违 |
| 调整税赋 | 修改该县/州的税赋定额 | 直接改变上缴额 | 影响其他县/州的分摊 |
| 资源调拨 | 从府库拨款支援 | 直接增加该县/州县库 | 减少府库 |
| 约谈施压 | 与下属对话，要求解释 | 下属承诺改进 + 好感度变化 | 过度施压 → 下属消极/请辞 |
| 弹劾免职 | 向上级推荐免除下属 | 需上级批准，下属被替换 | 新人需适应期，政治代价 |

> **设计原则**：越是强硬的干预手段，效果越直接但政治代价越高。
> 优秀的知府通过"建议"和"指令"即可引导下属，而非频繁约谈和弹劾。

## 5. 税赋管理

### 5.1 资金流向

```
各县/州税收（农业+徭役+商税）
    ↓ 按各县/州 remit_ratio 上缴
府级汇总
    ↓ 按省级定额上缴
省级（布政使司）

府库 = Σ(各县/州上缴) − 省级上缴 − 府衙行政开支 − 府级投资
```

### 5.2 税赋定额分配

**省级定额**：每年正月由布政使下达，总额根据府域类型和历史数据确定。

**知府配额计算**（向下分解到各县/州）：

```
各县/州农业税配额 = 该县/州在册耕地 × 0.5两/亩 × (1 + 水利等级×0.15)
                 × 农业税率 × 0.85（标准征收效率） × remit_ratio

各县/州徭役折银配额 = 该县/州在册村民人数 × 0.3两/人 × remit_ratio

说明：
  - 配额按"标准年"估算，不考虑农业适宜度（灾害风险由县/州自行应对）
  - 水利等级是已建成的实际设施，知府知悉并纳入配额
  - 省级定额按各县/州"理论产能"（土地×人口）比例分摊
```

**玩家（知府）分配**：玩家将省级定额分解到各县/州，设定每县/州的年度上缴目标。
系统按上述公式提供建议值，玩家可在合理范围内调整（过大→下属申诉，过小→府库填补差额）。

| 分配策略 | 说明 | 风险 |
|---------|------|------|
| 均摊 | 各县/州平均 | 弱县完不成，强县不满（太少了为何还要多交） |
| 按能力 | 强县/州多担 | 强县/州可能抵触，认为不公平 |
| 历史延续 | 沿用上任标准 | 安全但可能不合理 |
| 加码 | 总分配略高于省级定额 | 超额部分留府库，但增加下属压力 |

- 分配偏差过大 → 下属可能向巡抚申诉
- 总分配低于省级定额 → 府库需自行填补差额
- 连续两年未完成省级定额 → 考核严重减分

### 5.3 府衙行政开支

| 项目 | 年度开支 | 说明 |
|------|---------|------|
| 知府俸禄 | 200两 | 含正俸+养廉 |
| 通判俸禄 | 100两 | 财税副手 |
| 推官俸禄 | 80两 | 司法副手 |
| 幕僚束脩 | 60两 | 知府私聘 |
| 府衙书吏 | 80两 | 府级六房 |
| 衙署杂费 | 80两 | 办公/邮驿 |
| 府学经费 | 按等级 | 级1=120，级2=240，级3=480 |
| 驿道维护 | 按等级 | 级0=0，级1=100，级2=200 |

## 6. 基础建设

### 6.1 府级投资项目

| 项目 | 花费 | 工期 | 效果 |
|------|------|------|------|
| 扩建府学（1~3级） | 动态 | 4/6/10月 | 全府文教加成+5/级；府试选拔质量提升 |
| 修建跨县驿道（1~2级） | 动态 | 6/12月 | 信息传递加速；跨县商业+10% |
| 建设府级义仓 | 500两 | 即时 | 跨县赈灾时粮食调拨效率+30% |
| 赈灾拨款 | 可变 | 即时 | 从府库直接拨款至灾县/州 |
| 河道治理（1~2期） | 动态 | 12/18月 | 降低全府洪灾概率15%/25% |

> 府级投资影响全府，但花费远高于县级投资。详细公式见 06b。

### 6.2 全府年轻人才池（隐藏系统）

**设计意图**：在玩家不可见的后台维护一个全府年轻人才库，县学培养质量和府试选拔共同作用于人才输出，构成基础建设与人事管理之间的长期联动。

**初始化**（游戏开始时）：
- 遍历全府所有村庄，各村人口 × 3%，生成对应数量的年轻人才加入池
- 每人年龄在 19~22 之间随机分配

**年度补充**（每年三月自动执行）：
- 各村人口 × 1%，新生成年龄为 18 的人才加入池

**每位人才属性**：

| 属性 | 规则 |
|------|------|
| 籍贯 | 记录所属府、县、村 |
| 年龄 | 初始随机（见上），每年三月全员 +1 |
| 潜力值 | 初始随机 80~199，终身不变 |
| 能力值 | 初始在 [1, 潜力值/2] 区间随机；籍贯村有村塾则额外 +5 |

**县学培养**（每年自动结算）：
- 池内人才默认在所在县县学学习
- 年度能力值增量由所在县县学等级决定：

```
县学0级（未建） → +0
县学1级         → 随机 +1~2
县学2级         → 随机 +1~3
县学3级         → 随机 +2~4
```

**离池条件**：
- 府试入选（前100名）→ 进入仕途序列
- 年龄超过 35 岁未入选 → 移出池（归隐/务农）

### 6.3 府试（每年十月）

**机制**：
- 每年十月自动举行，全府才池所有人参加（游戏简化：跳过历史上的县试环节，直接参加府试）
- 按能力值排名取前 100 名

**府学等级影响**：
- 府学等级越高，选拔排名越准确
- 低等级府学时排名存在随机偏差，可能遗漏能力值较高的人才

**师生关系**：
- 当任知府与本次府试选出的 100 人建立隐性师生关系标记
- 这批人若后续进入仕途（成为官员 Agent），与该知府的初始好感度 +20，带"门生"标签
- 知府朝堂影响力的一部分由"门生数量 × 门生官职权重"计算（具体公式留给后续朝堂系统）

## 7. 人事管理

### 7.1 下属知县/知州属性

每位下属是完整的 AI Agent（复用官场 Agent 体系）：

```
AI 知县/知州属性：
├── 基础属性：智力、体魄（影响执政效率）
├── 性格三维：合群/理性/张扬（影响沟通方式）
├── 政治理念：社稷vs黎民 / 集权vs分权 / 务实vs理想
├── 执政风格：民本/政绩/保守/进取/圆滑
├── 对知府好感度：-99 ~ 99（影响服从度和汇报诚实度）
├── 能力评估：施政能力、廉洁程度、政治敏感度
└── 派系归属：影响其对政策的响应
```

### 7.2 年度评议（知府 → 知县/知州）

每年腊月，知府需对各下属撰写评议：

**评议维度**：
| 维度 | 权重 | 来源 |
|------|------|------|
| 税收完成率 | 30% | 客观数据 |
| 辖区稳定 | 20% | 治安/民心趋势 |
| 施政能力 | 20% | 知府主观评价 |
| 品行操守 | 15% | 知府主观评价 + 核查结果 |
| 服从配合 | 15% | 知府主观评价 |

**评议影响**：
- 优评 → 在三年大考中获得加分，下属好感度+
- 差评 → 大考减分，下属好感度-，可能导致关系恶化
- 虚假好评 → 如巡抚发现实际不符，损害知府信誉
- 虚假差评 → 下属可向上申诉，若查实则知府声望受损

### 7.3 人事建议权

知府可向上级提出人事建议（需巡抚批准）：

| 建议 | 条件 | 结果 |
|------|------|------|
| 推荐升迁 | 下属连续2年优评 | 巡抚审核后转吏部 |
| 建议平调 | 任意理由 | 巡抚审核后执行 |
| 建议免职 | 需有明确事由 | 巡抚调查后决定 |
| 举荐新人 | 有空缺时 | 可优先推荐府试出身的门生 |

- 人事建议的通过率受知府与巡抚关系、建议合理性、政治局势影响
- 推荐不称职者 → 长期声誉损失
- 打压有能力者 → 若被发现，清名受损

### 7.4 知府三年大考

知府任期三年接受大考，机制待后续单独设计。考核指标参见 §11。

## 8. 司法管理

### 8.1 推官职能

推官是知府的司法副手，负责初步筛查错判，辅助知府行使司法权。

**季度筛查**：每季度自动扫描下辖各县判决记录，标记 1~2 件疑似错判案件，并附简要线索：
- "证词矛盾，关键证人未传讯"
- "量刑明显偏重，参照律例有疑"
- "证据不足，主要依赖口供"

**推送给玩家决策**：

| 处理方式 | 说明 | 后果 |
|---------|------|------|
| 亲自审查 | 开启对话/调查事件，玩家深入研判 | 纠错则声望+；维持原判则无变化 |
| 委托推官 | AI 自行判断，可能纠错也可能维持 | 结果不确定，节省行动力 |
| 忽略 | 维持原判，不作处理 | 若被按察使复查发现错判 → 考核减分，"失察"记录 |

### 8.2 风险机制

- 错判未纠正，被按察使复查发现 → 知府考核减分，留"失察"记录
- 主动纠错 → 声望加成，推官好感+，按察使好感+
- 下属知县刻意判错（贪腐/私怨）而知府未察 → 视同知府失察，代价加重

> 具体案件类型、重审事件流程后续单独细化。

## 9. 上级关系与政治环境

### 9.1 直属上级

知府的直属上级为巡抚（二品），同时与布政使（财税）和按察使（司法）有工作交集。

| 角色 | 关注点 | 与知府的互动 |
|------|--------|-------------|
| 巡抚 | 全省稳定、危机处理、政绩 | 下达政策指令、灾害资源调配、年度评审 |
| 布政使 | 财税收入、经济发展 | 下达税赋定额、审计财务、拨款审批 |
| 按察使 | 司法公正、治安稳定 | 抽查案件、巡察治安、纠察失职 |

### 9.2 省级压力

每年正月，省级官员下达年度要求：

```
巡抚令：
  - 年度税赋定额：XX 两（布政使核定）
  - 施政重点：[农业增产 / 商业振兴 / 治安整顿 / 文教兴盛]（选1~2）
  - 特别事项：[若有] 征兵、大工程摊派、中央政策传达
```

- 施政重点与知府的实际投资方向不符 → 考核减分
- 完成施政重点 → 考核加分 + 巡抚好感提升
- 特别事项为强制执行，不服从后果严重

### 9.3 政治博弈

知府可进行以下政治活动：

| 活动 | 效果 | 风险 |
|------|------|------|
| 拜谒巡抚 | 了解上级意图、提升好感 | 过于频繁则显谄媚 |
| 上报政绩 | 提前展示成果 | 数据注水可能被核查 |
| 申请减免 | 灾年申请减免税赋定额 | 需提供充分证据 |
| 请求拨款 | 申请省级赈灾/工程拨款 | 竞争激烈，受好感和政绩影响 |
| 联合其他知府 | 联名上书请求政策支持 | 若失败可能牵连 |
| 举荐/弹劾 | 影响其他府/县人事 | 站队信号，有政治后果 |

## 10. 应急事件处理

### 10.1 事件分类

| 类别 | 典型情形 | 处理方式 |
|------|---------|---------|
| 县级上报 | 重大灾害、匪患、民变苗头 | 决定是否干预、拨款、调兵 |
| 跨县灾害 | 洪灾/疫情波及多县/州 | 府级应急流程（见 §10.2） |
| 省级指令 | 征兵摊派、大工程分摊、政策传达 | 分配到各县/州执行，不服从后果严重 |
| 司法事件 | 推官标记的疑似错判 | 见 §8 |
| 乡绅联合对抗 | 高税或工程征地引发联名上书 | 见 §10.3 |
| 官场事件 | 下属贪腐、考核争议、上级巡察 | 调查处理、应对巡察 |

### 10.2 跨县灾害应急

当灾害波及多县/州时，触发府级应急流程：

1. **灾情汇总**：各县/州上报灾情（可能延迟或低报）
2. **资源调配**：从府库拨款、从未受灾县调粮
3. **向上申报**：向巡抚申请赈灾支持
4. **后续追责**：决定是否追究知县/知州防灾不力

### 10.3 乡绅联合对抗

**触发条件**：
- 某县税赋定额超过该县历史均值 20% 以上，且连续2季度；或
- 知府推行府级工程（驿道/河道治理）征用土地

**事件形式**：一次对话/谈判事件，乡绅代表联名上书知府。

**处理选项**：

| 选项 | 效果 | 代价 |
|------|------|------|
| 让步 | 降低定额/缩小工程 | 民心+，但税收目标受影响 |
| 安抚 | 承诺补偿/政策优惠 | 消耗府库 |
| 强压 | 维持原策 | 乡绅好感-，后续触发概率升高 |

> 后续可扩展：乡绅向巡抚申诉、组织消极纳税等联动效果。

### 10.4 省级巡察

省级官员（按察使或巡抚委派）不定期巡察：

- 巡察可能发现：下属汇报不实、隐性问题、腐败
- 巡察结果影响知府考核——下属问题 = 知府失察
- 提前通知 vs 突击巡察——取决于与按察使的关系

## 11. 考核与评价

### 11.1 知府考核委员会（对齐 05 §2.2）

| 席位 | 人选 | 关注点 |
|------|------|--------|
| 本省巡抚 | ×1（直属上级） | 综合稳定与执行力 |
| 本省布政使 | ×1 | 财税完成率 |
| 本省按察使 | ×1 | 司法与治安 |
| 吏部侍郎 | ×1（中央代表） | 大局与政策执行 |
| 外省巡抚/布政使 | 随机×1 | 较客观 |

### 11.2 考核指标

**客观分（50%）**：

| 指标 | 权重 | 数据来源 |
|------|------|---------|
| 税赋完成率 | 30% | 全府三年累计上缴 ÷ 定额 |
| 辖区稳定 | 25% | 全府均民心/均治安变化趋势 |
| 经济发展 | 20% | 人口增长率、商业发展 |
| 施政重点完成 | 15% | 巡抚年度指令的执行情况 |
| 重大事故 | 10% | 任内是否发生暴动、大规模流民、瞒报 |

**主观分（50%）**：
- 与各委员好感度
- 委员个人偏好（同知县考核）
- 年终面谈表现
- 人事管理能力的评价

## 12. 时间系统与结算

### 12.1 时间节奏

**内部时钟**：月度（与县域一致，36个月/任期）。每月各县/州 AI 知县/知州独立决策并自动结算，玩家无需每月操作，但突发事件可在任何月份触发，要求即时响应。

**全年关键月份**：

| 月份 | 事项 |
|------|------|
| 正月 | 接收省级年度税赋定额；分解配额到各县/州；巡抚下达年度施政重点 |
| 二月 | 下属第一次年度汇报（指标档位 + 施政行动） |
| 三月 | 全府才池年度补充（各村人口×1%，年龄18入池）；县学培养年度结算（能力值增长） |
| 五月 | 下属第二次年度汇报；徭役折银征收（各县/州） |
| 八月 | 下属第三次年度汇报（含秋收情况） |
| 十月 | 府试：全府才池参加，选前100名，建立师生关系 |
| 十一月 | 下属第四次年度汇报（含全年税收完成情况） |
| 腊月 | 年度评议：知府对各下属撰写评语；人事建议提交；政治活动窗口 |

> 玩家可在任意月份执行：府级投资、下达指令、约谈下属、信息核查、政治活动（拜谒/上书等）。
> 推官的疑似错判推送每季自动触发，玩家可在当月或下月内处理。

### 12.2 结算节点说明

```
月度（持续）:
  各县/州 AI 知县/知州独立决策 + 物理引擎结算

正月:
  省级下达定额 → 玩家分解配额 → 各县/州收到年度上缴目标

二月 / 五月 / 八月 / 十一月:
  下属汇报生成 → 下辖县州总览页面刷新（档位数据）
  → 触发本期府级事件（跨县纠纷、省级指令、乡绅对抗等）

三月:
  才池年度补充 + 全府县学培养结算

十月:
  府试结算：按能力值排名，前100名入选，建立师生关系标记

腊月:
  年度评议窗口开放 → 玩家对各下属逐一评价
  → 人事建议提交巡抚

三年末（任期终）:
  大考（机制待设计）
```

### 12.3 信息时效

| 信息类型 | 延迟 | 说明 |
|---------|------|------|
| 财务数据 | 1个月 | 经通判汇总，下月可见 |
| 灾害报告 | 即时~1个月 | 重大灾害即时上报，小灾可能延迟 |
| 民心/治安 | 1个季度 | 季报中体现，月报为估算 |
| 人口变动 | 1年 | 年度户籍清查才知实数 |
| 下属真实行动 | 可能永不知 | 除非主动核查或他人举报 |

## 13. 核心矛盾与玩法

### 13.1 核心博弈

| 矛盾 | 两难 |
|------|------|
| 上缴 vs 发展 | 完成省级定额要求高税收，但压榨下属县/州影响长期发展 |
| 信任 vs 控制 | 信任下属让其自主发展效率高，但可能出问题；过度控制则效率低且关系差 |
| 公平 vs 效率 | 各县/州均衡分配看似公平，但集中资源到重点县效果更好 |
| 表象 vs 真相 | 报上来的数据可能失真，但频繁核查又耗费精力和关系 |
| 人情 vs 原则 | 下属有错，是包庇维护关系还是据实上报损害关系 |
| 短期 vs 长期 | 催逼税赋短期完成指标，但透支下属县/州的可持续性 |
| 投资 vs 即时 | 扩建府学、培育人才是长期投入，任期内未必见效 |

### 13.2 与知县/知州阶段的呼应

- 玩家在知县/知州阶段体验过"被上级催税"，现在反过来成为催税的人
- 玩家理解县级经济运作，能更好地判断下属汇报的合理性
- 玩家治理过的县/州现在由 AI 知县/知州管理——是否沿用你的政策？
- 从"被考核者"变为"考核者"——理解评议的双面性
- 知县阶段投资县学的效果，在知府阶段通过才池和府试显现

### 13.3 胜利路径

知府阶段没有单一最优策略。可能的风格：

| 风格 | 做法 | 考核优势 | 风险 |
|------|------|---------|------|
| 铁腕管理 | 高压催税、严格考核、频繁核查 | 税赋完成率高 | 下属怨恨、举报风险 |
| 放权信任 | 宽松定额、少干预、多支持 | 下属好感高、内部和谐 | 可能被欺骗、考核失控 |
| 重点突破 | 集中资源扶持1~2个县 | 有亮点政绩 | 其他县被忽视 |
| 八面玲珑 | 政治优先、搞好上下关系 | 主观分高 | 客观指标可能一般 |
| 育才兴教 | 大力投资府学、重视府试 | 门生网络扩展影响力 | 任期内效果滞后 |
